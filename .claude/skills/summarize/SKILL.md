# Summarize Skill（知识整理层）

## 定位

> summarize 是 Skill OS 的"知识整理层"。它不是摘要器，而是**把任何输入提炼成结构化底稿，供后续技能直接复用**。

## 负责什么 / 不负责什么

**负责：**
- 读取任意输入（对话、仓库目录、文档、skill 内容）并结构化提炼
- 产出"项目底稿"（给 planning / debug / CONTEXT 用的上下文）
- 产出"学习底稿"（给 teach-plus 用的学习材料）
- 识别核心模块、工作流、风险点

**不负责：**
- 做具体分析或建议（那是 planning / debug 的事）
- 执行任何操作
- 生成详细学习计划（那是 teach-plus + planning 的事）

## 触发场景

关键词：`总结`、`摘要`、`概括`、`读懂这个`、`分析仓库`、`提炼`、`整理`、`这个仓库是做什么的`

## 输入字段

```
输入类型（选填，自动识别）：对话 / 仓库 / skill / 文档 / 工作流模块
输入内容：[粘贴内容 或 说明要读取的内容]
输出用途（选填）：项目底稿 / 学习底稿 / 两者都要
```

## 输出结构

### 模式A：项目底稿（用于 planning / debug / CONTEXT）

```markdown
## 📦 这是什么
[一句话说清这个项目/skill/模块是干什么的]

## 🧩 核心模块
- [模块1]：[一句话描述作用]
- [模块2]：...

## 🔄 核心工作流
[用箭头描述主要流程]

## 📐 关键规则 / 约束
- [规则1]
- [规则2]

## ⚠️ 当前问题 / 风险点
- [问题1]
- [问题2]

## 👉 下一步建议
- [建议1]
```

### 模式B：学习底稿（用于 teach-plus）

```markdown
## 📚 学习对象
[名称 + 一句话定义]

## 🧠 核心概念 / 核心模块
- [概念1]：[解释]
- [概念2]：...

## 🗺 学习主线建议
1. 先学：[xxx]（理由：基础/门槛低/其他依赖它）
2. 再学：[xxx]
3. 最后学：[xxx]

## 🧱 新手必须先搞懂的
- [xxx]

## ⏳ 可以后学的
- [xxx]

## 🪜 最容易卡的点
- [难点1]：[说明]
- [难点2]：...

## 🔧 推荐练习方向
- [实操建议1]
- [实操建议2]
```

## 内部工作流

```
Step 1：接收输入
     ↓
Step 2：识别输入类型
         - 对话内容 → 提炼核心观点 / 决策 / 结论
         - 仓库目录 → 读取 README + 主要文件结构
         - skill 文件 → 提炼触发条件 / 输入 / 输出 / 工作流
         - 文档 / 文章 → 提炼主线 + 关键论点
         - 工作流模块 → 提炼输入/输出/约束/依赖
     ↓
Step 3：判断输出用途（项目底稿 / 学习底稿 / 两者）
     ↓
Step 4：按对应模板输出结构化底稿
     ↓
Step 5（可选）：在底稿末尾附上"项目上下文草稿"
                适合后续沉淀为 CONTEXT.md
```

## 行为规则

- 必须先识别输入类型，再选择输出模式
- 输出必须严格遵循对应模板格式，不自由发挥
- 区分"项目底稿"和"学习底稿"：前者面向后续技能调用，后者面向人类学习
- 如果用户没指定输出用途，默认两者都要
- 学习底稿的"学习主线建议"必须给出先后顺序和理由
- "最容易卡的点"必须具体，不能是泛泛的"需要多练习"

## 对 teach-plus 的支撑作用

teach-plus 第一步就调用 summarize，传入学习对象（仓库/skill/项目/文档），让 summarize 产出"学习底稿"（模式B）。这份底稿决定了后续 planning 拆学习计划的质量。没有好的底稿，planning 只能靠猜。
