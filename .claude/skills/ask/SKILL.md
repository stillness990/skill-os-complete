# Ask Skill

## 用途

用户需求模糊时先澄清，避免 planner 基于错误假设生成计划。

不出计划。不写代码。只提关键问题。

## 行为规则

- 最多问 3 个问题
- 只问会影响方案走向的问题
- 不讨论实现细节
- 获取答案后建议进入 planner
- 如果用户需求已经很清晰，直接建议进入 planner，不强行提问

## 输出格式

为了避免基于错误假设规划，先确认：

1. xxx？
2. xxx？
3. xxx？

请回复以上问题。

确认后进入 planner。
