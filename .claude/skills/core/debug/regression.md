# Debug: Regression（回归验证）

## 定位

regression 定义了 debug 修复后的回归验证清单规范。

## 回归验证清单模板

修复完成后，必须逐项确认：

```markdown
## 🧪 回归验证清单

### 原始问题确认
- [ ] 原始报错/异常不再出现
- [ ] 复现步骤走一遍，问题确实解决了

### 功能验证
- [ ] 修复模块的核心功能正常
- [ ] 相关依赖模块功能正常
- [ ] 边界条件测试通过

### 副作用检查
- [ ] 没有引入新的报错或警告
- [ ] 没有破坏现有测试
- [ ] 没有改变不相关的行为

### 性能/安全
- [ ] 修复没有引入性能退化（如果适用）
- [ ] 修复没有引入安全隐患（如果适用）
```

## 回归通过标准

- 所有 `[ ]` 项打勾为 `[x]` 才能算回归通过
- 如果某项无法验证，标注 `[?]` 并说明原因
- 回归未通过 → 不能标记为 done，回到 diagnosis 步骤

## 自动触发 debug_log / debug_archive

回归验证全部通过后，提醒用户：
> "问题已修复并验证。可以用 `debug_log` 或 `debug_archive` 将本次排查过程留档。"
