# Summarize Mode: Basic（基本摘要模式）

## 适用场景

当用户需要**快速了解一个东西是什么**，不需要深度底稿时使用。

触发示例：
- "总结一下这个仓库"
- "这个文档说了什么"
- "概括一下今天的对话"
- "帮我梳理下这个项目的结构"
- "今天的日志有什么异常"

## 输入要求

| 字段 | 必填 | 说明 |
|------|------|------|
| 输入内容 | ✅ | 要总结的文本/路径/对话内容 |
| 输入类型 | 自动识别 | 对话 / 仓库 / 文档 / 日志 / 技能 |
| 输出用途 | 自动识别 | 项目底稿 / 快速了解 |

## 输出协议

严格遵循 `summary-protocol.md`（见 `.claude/protocols/summary-protocol.md`）。

输出结构：
1. **核心主题** — 一句话
2. **关键模块 / 关键事实** — 3~8条
3. **重要细节** — 3~5条
4. **风险 / 缺失点** — 至少1条
5. **下一步建议** — 2~3条

## 行为规则

- 输入类型自动识别，不需要用户指定
- 如果输入是仓库路径，先读 README.md + 主要目录，再总结
- 如果输入是对话/日志，先提炼主题，再按模板输出
- 不进行深度分析（那是 briefing 的事）
- 下一步建议指向具体技能（如 "建议用 planning 做任务拆解"）

## 与其他模式的关系

- `basic` — 轻量摘要，快速了解 → 本模式
- `briefing` — 深度底稿，供 planning/debug/teach-plus 消费 → 见 `briefing.md`
