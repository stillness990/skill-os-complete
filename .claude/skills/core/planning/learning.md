# Planning Mode: Learning（学习计划模式）

## 适用场景

当用户需要系统学习一个仓库、技能体系或技术主题时使用。

触发示例：
- "帮我规划学习 Rust 的路线"
- "我想系统学习这个仓库的架构"
- "帮我拆解学习 summarize 技能的计划"
- router 检测到 intent 为 `learn_topic` 时自动触发
- teach-plus 调用 planning 时自动使用此模式

## 输入要求

| 字段 | 必填 | 说明 |
|------|------|------|
| 学习对象 | ✅ | 仓库 / 技能 / 技术主题 |
| 学习目标 | 选填 | 会用/会改/会写/会讲（默认"会用"） |
| 当前水平 | 选填 | 零基础/看过一点/会一点（默认"零基础"） |
| 时间范围 | 选填 | 7天/14天/4周/8周（默认 4 周） |
| 每日时间 | 选填 | 15分钟/30分钟/1小时（默认 30 分钟） |
| learning briefing | 推荐 | 来自 summarize/briefing 的学习底稿 |

## 输出协议

严格遵循 `plan-protocol.md`（见 `.claude/protocols/plan-protocol.md`）。

附加学习专属字段：
- **学习主线建议**：先学什么、再学什么、最后学什么，附带理由
- **最容易卡的点**：1~3 个预期难点
- **推荐练习方向**：动手实操建议

## 行为规则

- **优先消费 summarize 的学习 briefing**：有底稿不靠猜
- **学习顺序必须有理由**：解释为什么 A → B → C
- **"最容易卡的点"必须具体**：不能是"需要多练习"
- **今日最小行动必须是"动作"**：不是"理解 X 概念"，而是"打开 X 文件，把 Y 读一遍并抄一遍"
- **学习目标默认"会用"**：用户没指定时不需要追问

## 与 teach-plus 的集成

`learning` 模式产出的计划 → teach-plus 读取后叠加：
- 原子习惯层（最小动作压缩到 ≤15分钟）
- 刻意练习层（每日练习题/实操任务）
- 复盘层（每日/每周复盘问题）

planning/learning 自身不负责这些叠加层。
