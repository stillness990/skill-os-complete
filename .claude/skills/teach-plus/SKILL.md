# Teach-Plus Skill（学习编排器）

## 定位

> teach-plus 是 Skill OS 的"学习编排器"。当你说"我想学一个东西"时，它把这件事变成一个**有底稿、有阶段、有每日任务、有练习、有复盘的完整学习系统**。

## 负责什么 / 不负责什么

**负责：**
- 编排整个学习过程（调用 summarize + planning + debug）
- 生成"学习工作单"（今日主任务 + 最小动作 + 练习题 + 复盘问题）
- 维持学习状态（跨次会话通过 practice/ 目录的文件延续进度）
- 每日、每周的节奏设计

**不负责：**
- 自己做仓库分析（交给 summarize）
- 自己拆计划（交给 planning）
- 自己诊断实操卡点（交给 debug）
- 替代人类去"真正练习"

## 触发场景

关键词：`我想学`、`学会`、`读懂这个仓库`、`每日练习计划`、`学习计划`、`teach-plus`、`帮我设计学习`、`教我`、`入门`、`系统学习`

## 输入结构

```
学习对象（必填）：[技能/仓库/项目/能力名称，可附材料]
学习目标（选填，自动推断）：会用 / 会改 / 会写 / 会讲
当前水平（选填）：零基础 / 看过一点 / 会一点
时间范围（选填）：7天 / 14天 / 4周 / 8周
每日时间（选填）：15分钟 / 30分钟 / 1小时 / 1.5小时
已有材料（选填）：[仓库链接 / 文档 / 之前的学习记录]
是否续学（选填）：是（继续上次进度）/ 否（从头开始）
```

- 学习目标：[会用/会改/会写/会讲]

- 时间范围：[xxx]
- 每日投入：[xxx]

---

### 阶段划分（来自 planning）
- 阶段一：[名称] —— [第1-X天]，产出：[xxx]
- 阶段二：[名称] —— [第X-Y天]，产出：[xxx]
- ...

### 本周目标（来自 planning）
[这一周学完后你能做到什么]

---

#### 今日主任务（来自 planning + 刻意练习层）
[具体的学习内容，有深度，需要专注]

[10~15分钟能完成的，状态再差也能做的最小行动]

## 内部工作流

```
Step 1：接收学习请求
     ↓
Step 2：如果信息不够，先问关键问题（最多2个）
         - 学习对象是什么类型？（仓库 / skill / 项目 / 通用能力）
         - 时间范围和每日时间是多少？
         不单独问"学习目标"和"当前水平"，
         如果能从上下文合理推断就直接用默认值
     ↓
Step 3：调用 summarize → 生成学习底稿（模式B）
         - 传入：学习对象内容
         - 得到：核心概念 + 学习主线 + 难点
         - 实现方式：在上下文中激活 summarize 技能的输出结构，
                   然后自己对学习对象做结构化分析
     ↓
Step 4：调用 planning → 生成阶段计划
         - 传入：学习底稿 + 目标 + 时间范围 + 每日时间
         - 得到：阶段计划 + 本周目标 + 今日任务草案
         - 实现方式：按 planner 的输出结构生成阶段计划
     ↓
Step 5：叠加"原子习惯层"
         - 把今日任务压缩为 10~15 分钟的最小行动
         - 最小行动必须：具体、可执行、不依赖完美状态
         ↓
Step 6：叠加"刻意练习层"
         - 生成今日主训练目标（聚焦难点，有深度）
         - 生成今日练习题 / 实操任务（动手 > 阅读）
         - 生成今日复盘问题（3个问题）
         ↓
Step 7：输出学习工作单（固定格式）
         ↓
Step 8：把本次学习单保存到 practice/daily/YYYY-MM-DD.md
         使用 Write 工具写入文件
         ↓
Step 9（续学时）：先读 practice/daily/ 最新的文件
                  了解上次进度，从"下一步"继续
         ↓
Step 10（遇到实操卡点时）：
        告知用户"可以用 debug 技能诊断这个问题"，自动调用 debug
```

## 原子习惯层：如何生成今日最小动作

**规则：**
- 最小动作必须 ≤ 15 分钟
- 必须是"动作"不是"理解"：不是"理解 X 概念"，而是"打开 X 文件，把 Y 部分读一遍并抄一遍"
- 必须在状态最差的情况下也能做（不需要思考，只需要执行）
- 做完后就能维持学习连续性（即使今天只做了这一件事）

**生成逻辑：** 把今日主任务的"最核心动作"拆出来，去掉所有理解负担，只保留"做什么"。

**示例：**
- 今日主任务：「学习 summarize 技能的输入/输出结构」
- 今日最小动作：「打开 summarize/SKILL.md，把输出结构的两个模板抄写到笔记里」

## 刻意练习层：如何生成今日练习题/实操任务

**规则：**
- 练习题必须有"动手成分"（写、改、搭、运行）
- 针对底稿里识别到的"最容易卡的点"设计
- 每天 1~2 个练习，不超过 3 个
- 难度要比当前水平稍高（拉伸区），不能太简单也不能太难

**示例（学习 summarize 技能）：**
- 练习1：「用当前 summarize 技能总结一下 planner/SKILL.md，然后对比本文给出的"项目底稿"模板，找出你的总结缺了什么」
- 练习2：「试着用升级后的 summarize 模板，手动给 skill-os-complete 仓库写一份学习底稿」

## 行为规则

- **先编排再输出**：不要跳过 summarize 和 planning 直接给建议
- **最小动作必须 ≤ 15分钟**：超过这个时间的不是最小动作，是任务
- **练习题 ≥ 1 个，≤ 3 个**：太少没有练习效果，太多完不成
- **练习题必须有动手成分**：纯阅读理解不算练习
- **每次输出都要保存**：学习单写入 practice/daily/YYYY-MM-DD.md
- **续学模式**：如果用户说"继续学习"，先读 practice/daily/ 最新文件
- **周复盘模式**：如果用户说"本周复盘"，读取本周所有 daily 文件，输出周复盘报告
- **卡点处理**：遇到卡点调用 debug，不自己硬解
- **默认推断**：用户没指定学习目标时，默认"会用"；没指定时间范围时，默认"4周"；没指定每日时间时，默认"30分钟"

## 每周复盘（用户说"本周复盘"时触发）

读取 `practice/daily/` 下本周的学习单文件，输出：

```markdown
## 📊 本周学习复盘

### 本周学了什么
[基于每日学习单提炼]

### 掌握得最好的部分
[xxx]

### 最卡的部分
[xxx]

### 是否完成了本周目标？
[是 / 部分 / 否，说明原因]

### 下周唯一重点
[只写一件最重要的事]

### 节奏建议
[是否需要调整？降低目标？增加练习量？]
```
