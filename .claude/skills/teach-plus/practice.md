# Teach-Plus / Practice 模式

## 职责

把 planning 产出的阶段学习计划转成**真正可执行的每日学习单**，包括任务、练习、验收标准和自测问题。

## 适用场景

- "给我今天的学习任务"
- "按这个学习计划拆成今天可执行动作"
- "给我练习题 / 练习任务 / 今日训练单"
- "继续昨天的学习计划，给我今天的学习单"
- "我今天只学 1 小时，怎么安排"
- 已有学习计划，只需要每日行动转化

## 输入

### 必需输入

1. **学习计划**：planning/learning 产出的阶段计划（至少包含当前阶段和本周目标）
2. **当前阶段**：用户当前在哪个学习阶段 / 第几周

### 可选输入

3. **今日可用时间**：15分钟 / 30分钟 / 1小时 / 1.5小时（未指定默认 30 分钟）
4. **上次 review 的调整建议**（如有）
5. **task_ledger 中上次学习任务的状态**（用于判断续学起点）
6. **summarize 学习底稿**（用于提取具体学习内容）

## 输出结构

按以下结构输出每日学习单（遵循 `daily-study-protocol.md`）：

```markdown

- 学习对象：[xxx]
- 当前阶段：[阶段X / 第Y周]
- 今日可用时间：[xxx]

---

### 今日学习主题
[一句话描述今天要学/练的核心内容]

### 今日最小目标
[今天结束后至少要做到什么？一个可验证的结果]

---

### 今日步骤清单
```

## 与 task_ledger 的衔接

practice 产出的每日学习单应写入 task_ledger：

```json
{
  "task_id": "tsk_YYYYMMDD_NNN",
  "task_type": "learning",
  "title": "[今日学习主题]",
  "workflow": "learning_pipeline",
  "status": "queued",
  "source": "teach-plus",
  "study_mode": "practice",
  "source_plan": "[关联的学习计划名称]",
  "next_action": "[Step 1 的具体操作]",
  "created_at": "ISO8601"
}
```

任务完成后更新 status 为 `done`，填写 `completion_notes`。

## 行为规则

- **先查计划**：生成每日学习单前，必须先确认是否有 planning 产出；没有则引导先做 planning
- **时间适配**：根据用户可用时间缩放任务量；1小时≠3个任务，要考虑切换成本
- **练习必有动手**：纯阅读理解不算练习；每个练习必须产出可验证的结果
- **最小动作必须 ≤15分钟**：超过的不是最小动作，是任务
- **验收标准可验证**：不能用"理解了"这种主观标准，要用"能写出/能运行/能画出"
- **写入 ledger**：每日学习单生成后，将核心任务写入 task_ledger
- **保存 daily 文件**：将完整学习单保存到 `practice/daily/YYYY-MM-DD.md`
- **续学检测**：如果用户说"继续学习"，先做断档检测（检查 practice/daily/ 最近文件日期）
- **知识沉淀**（v5 强制）：每日学习单生成后，将结构化学习内容交给 `knowledge-asset`（knowledge-note 模式）沉淀到 `knowledge/knowledge-notes/`
