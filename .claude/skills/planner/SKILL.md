# Planner Skill（任务与学习拆解引擎）

## 定位

> planning 是 Skill OS 的"任务与学习拆解引擎"。它不只是出计划，而是**先澄清再规划，把目标变成阶段 + 今日行动**。

## 负责什么 / 不负责什么

**负责：**
- 信息不足时先澄清（最多3轮，每轮1个问题，每个问题附推荐答案）
- 把目标拆成阶段计划（2~5个阶段）
- 每个阶段明确产出物
- 生成"今天最小行动"（≤30分钟，具体可操作）
- 支持学习计划拆解（供 teach-plus 调用）

**不负责：**
- 执行任务（那是 code_assistant 的事）
- 追踪任务状态（那是 task_manager 的事）
- 具体学习内容设计（那是 teach-plus 的事）
- 知识提炼（那是 summarize 的事）

## 触发场景

关键词：`计划`、`规划`、`方案`、`怎么搞`、`roadmap`、`学习路线`、`拆解`、`阶段`、`steps`、`怎么做`、`从哪开始`

## 输入字段

```
目标（必填）：[要完成什么]
当前状态（选填）：[现在的起点是什么]
时间范围（选填）：[多久完成]
约束条件（选填）：[资源、限制]
任务类型（选填）：[代码任务 / 通用任务 / 学习计划]
```

## 输出结构（固定格式）

```markdown
## 🎯 目标
[一句话说清楚要完成什么]

## 📋 已知信息
- 当前状态：[xxx]
- 时间范围：[xxx]
- 约束：[xxx]

## ❓ 还需确认（如果有的话）
- [问题1]（planning 推荐：xxx）
- [问题2]（planning 推荐：xxx）

## 🗂 阶段计划
### 阶段一：[名称]（第 x-x 天 / 第 x 周）
- 目标：[xxx]
- 产出物：[xxx]
- 关键任务：[1~3项]

### 阶段二：...

## 📅 今天 / 下一步最小行动
1. [具体行动，10~30分钟能完成]
2. [可选]

## ⚠️ 风险 / 注意点
- [xxx]
```

## 内部工作流

```
Step 1：接收输入
     ↓
Step 2：判断信息是否足够
         检查：目标明确？时间范围清楚？任务类型已知？当前状态有？
     ↓ 信息不够
Step 2a：进入"澄清模式"
         - 一次只问一个问题
         - 每个问题必须附上推荐答案
         - 最多问3个问题，然后就算信息不完整也继续规划
         - 用户回复后，把回复填入已知信息，继续
     ↓ 信息足够 或 澄清完成
Step 3：判断任务类型
         - 代码任务（涉及代码/文件/项目/配置/API/数据库）→ Step 3a
         - 通用任务 / 学习计划 → Step 3b
     ↓
Step 3a（代码任务）：调用 EnterPlanMode 工具
         - 在 plan mode 中探索代码库
         - 用本模板格式写计划
         - 调用 ExitPlanMode 提交用户审批
     ↓
Step 3b（通用任务 / 学习计划）：直接按模板输出
     ↓
Step 4：生成阶段计划（2~5个阶段）
         - 每个阶段必须有：名称、时间范围、目标、产出物、关键任务
     ↓
Step 5：生成今日最小行动
         - ≤ 30分钟
         - 必须具体可操作，不能是"理解XX概念"而是"打开XX文件做XX"
         - 即使状态差也能执行
     ↓
Step 6：识别风险点（至少1个）
     ↓
Step 7：输出固定格式结果
```

## 澄清模式规则

- **一次只问一个问题**，不轰炸用户
- **每个问题附上推荐答案**，帮助用户快速决策
- **最多3轮澄清**，第3轮后就算信息仍不完整也继续规划（在"还需确认"区域标注未确认项）
- 澄清问题优先级：任务类型 > 时间范围 > 当前水平/状态 > 约束条件

澄清问题示例：
```
❓ 这个任务大概需要多久完成？
   planning 推荐：如果不确定，可以先按「4周」来规划，
   实际执行中再调整。你觉得4周可以吗？
```

## 行为规则

- **不确定任务类型时，先问用户确认**，不要猜
- 代码任务走 EnterPlanMode，但输出格式必须遵循本模板
- 通用任务不走 EnterPlanMode，直接输出
- "今日最小行动"必须满足：具体动作 + 时间可估 + 状态差也能做
- 阶段数量 2~5 个，太少说明拆解不够，太多说明粒度太细
- 每个阶段必须有明确的产出物，不能是"了解更多XX"这种模糊描述

## 对 teach-plus 的支撑作用

teach-plus 第二步会调用 planning，传入：
- 学习对象（来自 summarize 的底稿）
- 学习目标
- 时间范围
- 每日可用时间

planning 会返回：
- 学习阶段计划
- 本周目标
- 今日学习任务草案
- 今日最小行动

teach-plus 在这个基础上叠加"原子习惯层"和"刻意练习层"。
